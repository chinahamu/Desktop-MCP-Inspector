# Packaging and release guide

This guide covers the v0.1 packaging flow for Desktop MCP Inspector.

## Package outputs

The repository uses CPack from the top-level `CMakeLists.txt` to define cross-platform package metadata and install rules.

| Platform | Primary artifact | Fallback artifact |
| --- | --- | --- |
| Windows | NSIS installer | ZIP archive |
| macOS | DMG | TGZ archive |
| Linux | AppImage and DEB | TGZ archive |

The application target is `desktop_mcp_inspector`, and the installed executable name is `DesktopMCPInspector`.

## Local packaging

Build a release configuration first:

```bash
cmake --preset release -DDMI_BUILD_TESTS=OFF
cmake --build --preset release --parallel
cmake --install build/release --prefix build/release/install
```

Then run CPack with the generator for your platform.

Windows:

```powershell
cpack --config build/release/CPackConfig.cmake -G NSIS -B package
cpack --config build/release/CPackConfig.cmake -G ZIP -B package
```

macOS:

```bash
cpack --config build/release/CPackConfig.cmake -G DragNDrop -B package
cpack --config build/release/CPackConfig.cmake -G TGZ -B package
```

Linux:

```bash
cpack --config build/release/CPackConfig.cmake -G DEB -B package
cpack --config build/release/CPackConfig.cmake -G TGZ -B package
bash scripts/package-linux-appimage.sh build/release package
```

## GitHub release upload

The `Release Packages` workflow runs when one of these events occurs:

- a tag matching `v*.*.*` is pushed
- a GitHub Release is published
- the workflow is manually dispatched

Each platform job uploads its packages as workflow artifacts. For tag and Release events, the same files are uploaded to the GitHub Release by `softprops/action-gh-release`.

## Tooling notes

- Windows installer generation requires NSIS. The workflow installs it with Chocolatey.
- macOS DMG generation uses CPack `DragNDrop`.
- Linux DEB generation uses CPack `DEB`.
- Linux AppImage generation uses `linuxdeploy` with the Qt plugin. The helper script downloads the AppImage tooling at release time.
- Qt deployment is generated by Qt's CMake deploy helper on Windows and macOS when the installed Qt version exposes `qt_generate_deploy_app_script`.

## Release naming

Use semantic version tags such as `v0.1.0`. CPack package names include the project version from CMake and the platform name, for example:

```text
DesktopMCPInspector-0.1.0-Windows-AMD64.exe
DesktopMCPInspector-0.1.0-Darwin-arm64.dmg
DesktopMCPInspector-0.1.0-Linux-x86_64.deb
DesktopMCPInspector.AppImage
```

Actual CPU architecture names depend on the runner and CMake's `CMAKE_SYSTEM_PROCESSOR` value.
