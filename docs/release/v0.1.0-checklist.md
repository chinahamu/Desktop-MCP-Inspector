# v0.1.0 release checklist

Use this checklist before publishing the first public preview package.

## Scope

- [x] Confirm the release target is `v0.1.0`.
- [x] Confirm the release branch is created from the latest `main`.
- [x] Confirm all required Phase 0 through MVP tasks are merged.
- [x] Confirm T150 through T159 release operations are merged.

## Build and test

- [x] Windows `Build and Test` workflow is green on `main`.
- [x] Release workflow succeeds for Windows installer and ZIP.
- [x] Release workflow succeeds for macOS DMG and TGZ.
- [x] Release workflow succeeds for Linux AppImage, DEB, and TGZ.
- [x] Local smoke test: application starts on Windows.
- [x] Local smoke test: sample stdio MCP server connects.
- [x] Local smoke test: `initialize` succeeds.
- [x] Local smoke test: `tools/list` shows the sample `echo` tool.
- [x] Local smoke test: `tools/call` returns echo content.

## Documentation

- [x] README quick start is accurate for the release.
- [x] README screenshot is updated from the placeholder to a real release screenshot if available.
- [x] Sample MCP server guide is verified.
- [x] Packaging guide is verified against the release workflow.
- [x] README Japanese/English policy is followed for user-facing changes.
- [x] Security policy still matches the reporting process.

## License and notices

- [x] `LICENSE` is Apache-2.0.
- [x] README license section points to `LICENSE`.
- [x] `docs/license-audit.md` is reviewed.
- [x] Qt license obligations are checked for the chosen distribution mode.
- [x] Test-only dependencies are not redistributed unless their notices are included.
- [x] Third-party runtime dependencies bundled in installers are documented.

## Release notes

- [ ] Summarize supported platforms.
- [ ] Summarize known limitations.
- [ ] Include sample MCP server instructions.
- [ ] Include SHA256 checksums for release assets.
- [ ] Mention that early preview packages may require manual Qt/runtime troubleshooting.

## Publish

- [ ] Create and push tag `v0.1.0`.
- [ ] Confirm release assets are uploaded automatically.
- [ ] Download and smoke-test each uploaded artifact.
- [ ] Publish GitHub Release notes.
- [ ] Open follow-up issues for packaging or documentation gaps found during release.
